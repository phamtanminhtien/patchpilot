# PatchPilot Agent Instructions

Read before editing:

1. `docs/project-rules.md` - locked rules, highest authority.
2. `docs/product-spec.md` - active product scope/plan.
3. `docs/design-language.md` - required when creating or changing UI.
4. Task-related files.

If the task conflicts with `docs/project-rules.md`, stop and ask for explicit approval.

For every change: keep active product scope, update docs first for behavior/API/data/stack/structure/scope changes, use patch-first edits, run the narrowest verification, and report files plus results.
